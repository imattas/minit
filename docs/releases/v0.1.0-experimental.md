# v0.1.0-experimental

`v0.1.0-experimental` is the first VM-proven milestone for `minit`. It is intended for development and evaluation in QEMU/initramfs environments only.

This release is not daily-driver-ready.

## Verified Capabilities

The full local release gate proves:

- `minitd` boots as PID 1 in normal mode inside QEMU.
- cgroups v2 is mounted and used for service ownership.
- service definitions load from TOML files under `/etc/minit/services`.
- `minitd` exposes a Unix control socket at `/run/minit/minitd.sock`.
- `minitctl` can connect to the control socket.
- `minitctl status <unit>` reports loaded unit state.
- `minitctl list` reports a compact unit list.
- `minitctl start <unit>` starts a VM-safe service.
- `minitctl stop <unit>` stops a service and marks it inactive.
- `minitctl restart <unit>` stops and starts a service again.
- `minitctl explain <unit>` reports graph and lifecycle context.
- `minitctl events` reports the recent diagnostic event buffer.
- stopped service cgroups are removed after they drain.
- restart policy, backoff, and rate limiting are covered by unit and VM tests.
- target units can start dependency graphs in normal mode.
- mount units and swap units are modeled and deactivated during shutdown.
- optional mount failures can fail without taking down the boot.
- release artifacts and checksums are generated locally.
- long-running services, repeated boot/shutdown loops, stuck service stop, and stuck shutdown escalation are covered by QEMU smokes.
- strict unit parsing, UID/GID switching, clean environment handling, cgroups v2 resources, unsupported-security fail-closed behavior, and control socket permissions are implemented.
- managed services are stopped before VM poweroff during the shutdown smoke.
- rescue/initramfs mode can mount early filesystems, start a shell/getty path, reap children, and power off in the autoshutdown smoke.

## Release Gate

Run the source-only gate:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File tools\verify-release.ps1
```

Run the full VM gate:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File tools\verify-release.ps1 -Kernel C:\minit-vm\bzImage -BusyBoxPath C:\minit-vm\busybox
```

The full VM gate builds a normal-mode initramfs and runs these smokes:

- status smoke for `sshd`
- lifecycle restart smoke for `demo-sleep`
- cgroup cleanup smoke for `demo-sleep`
- restart-policy smoke for `crashy`
- shutdown-stop smoke for `demo-sleep`
- target boot smoke for `multi-user.target`
- mount success and optional mount failure smokes
- shutdown mount deactivation smoke
- diagnostic events smoke
- long-running service smoke
- repeated boot/shutdown loop smoke
- stuck stop and stuck shutdown escalation smokes

## Known Limitations

The following gaps intentionally block daily-driver claims:

- no parallel boot scheduler.
- no persistent journal replacement.
- broader distro install validation is still limited to the included profile and initramfs/QEMU smokes.
- fuzz harnesses exist, but a fuzzing gate is not yet part of CI.
- release artifacts are generated locally; hosted release signing is not configured yet.
- seccomp and filesystem sandboxing options are not implemented.

## Supported Evaluation Mode

Use this release only in a disposable VM or initramfs smoke environment. Do not install it as a host system's primary init.

## Next Release

`v0.2` should focus on broader validation and release hardening:

- CI fuzz/audit gates
- longer VM stress runs
- reproducible distro VM validation
- signed hosted release artifacts
- additional service sandboxing
