# v0.2.0-experimental

`v0.2.0-experimental` is the release-hardening and distro-validation milestone for `minit`. It is intended for development and evaluation in disposable QEMU environments only.

This release is not daily-driver-ready.

## New Since v0.1.0

- Added `minitctl list` for compact unit listing.
- Added a VM smoke for `minitctl list`.
- Added GitHub security workflow with dependency audit and bounded fuzz smokes.
- Added local security verification through `tools/verify-security.ps1`.
- Added extended VM stress mode with `-ExtendedVmStress -StressBootCount`.
- Added reproducible Alpine minirootfs VM validation.
- Added `config/profiles/alpine-minirootfs`.
- Added release workflow for tagged builds.
- Added GitHub artifact attestations for release artifacts.
- Added optional local GPG detached signing through `tools/sign-release.ps1`.
- Updated release, profile, and security documentation to match the verified implementation.

## Verified Capabilities

The full local release gate proves:

- `minitd` boots as PID 1 in normal mode inside QEMU.
- cgroups v2 is mounted and used for service ownership.
- service definitions load from TOML files under `/etc/minit/services`.
- `minitd` exposes a Unix control socket at `/run/minit/minitd.sock`.
- `minitctl status`, `list`, `start`, `stop`, `restart`, `explain`, and `events` work against the control socket.
- stopped service cgroups are removed after they drain.
- restart policy, backoff, and rate limiting are covered by unit and VM tests.
- target units can start dependency graphs in normal mode.
- mount units and swap units are modeled and deactivated during shutdown.
- optional mount failures can fail without taking down the boot.
- long-running services, repeated boot/shutdown loops, stuck service stop, and stuck shutdown escalation are covered by QEMU smokes.
- strict unit parsing, UID/GID switching, clean environment handling, cgroups v2 resources, unsupported-security fail-closed behavior, and control socket permissions are implemented.
- release artifacts and checksums are generated locally.
- Alpine minirootfs validation boots a real distro root filesystem in QEMU.

## Release Gate

Run the full VM gate:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File tools\verify-release.ps1 -Kernel C:\minit-vm\bzImage -BusyBoxPath C:\minit-vm\busybox -VmTimeoutSeconds 30
```

Run the Alpine minirootfs distro-rootfs gate:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File tools\vm\verify-alpine-minirootfs.ps1 -Kernel C:\minit-vm\bzImage
```

Run the local security gate:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File tools\verify-security.ps1
```

On Windows, local fuzz smokes are skipped unless the LLVM ASAN runtime is installed. The GitHub security workflow runs fuzz smokes on Ubuntu.

## Known Limitations

- No parallel boot scheduler yet.
- No persistent journal replacement.
- No device manager, network manager, or user session manager.
- No seccomp support yet.
- Filesystem sandboxing options remain fail-closed unless implemented.
- Distro validation is still QEMU/initramfs based, not a full disk install.

## Supported Evaluation Mode

Use this release only in disposable VMs or recoverable test systems. Do not install it as a host system's primary init.
